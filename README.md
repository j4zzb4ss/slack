# Static Timing Analysis: Slack (Временной запас)

**Slack** (Временной запас) — это ключевая метрика в статическом временном анализе (STA), которая показывает разницу между требуемым временем прибытия сигнала и его фактическим временем прибытия.

## Основная формула
Slack рассчитывается как:
`Slack = Required Time - Arrival Time`

* **Arrival Time (Время прибытия):** Время, за которое сигнал проходит по пути от выхода одного триггера до входа следующего.
* **Required Time (Требуемое время):** Момент времени, определяемый тактовым сигналом, к которому данные должны быть готовы и стабильны.

---

## Значения Slack

1.  **Positive Slack (> 0):** Сигнал пришел раньше дедлайна. Схема работает корректно на данной частоте.
2.  **Zero Slack (= 0):** Сигнал пришел точно в срок. Запас надежности отсутствует.
3.  **Negative Slack (< 0):** Сигнал опоздал (Timing Violation). Схема не будет работать корректно на целевой частоте.

---

## Setup Slack и Hold Slack

Для стабильной работы триггера необходимо соблюдение двух условий: времени предустановки (Setup) и времени удержания (Hold).



### 1. Setup Slack (Время предустановки)
Проверяет, успевает ли сигнал закрепиться на входе триггера до активного фронта тактового сигнала.
* **Нарушение:** Сигнал идет слишком долго (длинный путь комбинационной логики).
* **Как исправить:**
    * Снизить тактовую частоту.
    * Оптимизировать комбинационную логику (уменьшить количество вентилей между триггерами).
    * Использовать более быстрые элементы (Cells).

### 2. Hold Slack (Время удержания)
Проверяет, остается ли сигнал стабильным достаточное время после фронта тактового сигнала, чтобы данные не "проскочили" насквозь.
* **Нарушение:** Сигнал дошел слишком быстро (слишком короткий путь).
* **Как исправить:**
    * Добавить буферы задержки (Delay lines) на пути данных.
    * **Важно:** Изменение частоты тактового сигнала никак не влияет на Hold-анализ.

## Сравнительная таблица

| Характеристика | Setup Slack | Hold Slack |
| :--- | :--- | :--- |
| **Критический путь** | Самый длинный путь данных | Самый короткий путь данных |
| **Зависимость от частоты** | Напрямую зависит (T_clk) | Не зависит |
| **Влияние на чип** | Можно запустить на меньшей частоте | Чип неработоспособен (Hard failure) |
| **Способ исправления** | Ускорение логики / Снижение частоты | Замедление логики (вставка буферов) |
